# Nashville Housing Data Cleaning Project

## Overview

This project focuses on cleaning the Nashville housing dataset using Oracle SQL. The aim is to clean and structure the raw data to make it ready for analysis while demonstrating SQL proficiency.

## Dataset

The dataset includes various attributes of housing properties in Nashville, such as:

- **Parcel ID**: Unique identifier for properties.
- **Property Address**: The address of the property.
- **Owner Name**: Name of the owner.
- **Sale Date**: Date of sale.
- **Sale Price**: Transaction amount.
- **Property Type**: Type of the property (e.g., Residential, Commercial).
- Other relevant details about the properties.

## Objectives

1. Identify and handle missing or inconsistent data.
2. Remove duplicate entries.
3. Standardize formats for dates and text fields.
4. Enhance the dataset by creating additional columns.
5. Optimize the dataset for downstream analysis.

## Results

After cleaning, the dataset:

- Contains no duplicates or null values in critical columns.
- Has consistent formats for dates and text fields.
- Includes new fields (e.g., SaleYear) to enhance analysis capabilities.
- Is well-structured and ready for analytical use.

## Tools Used

- **Oracle SQL**: For writing and executing the cleaning queries.
- **Database Management System**: Oracle Database.

## Usage Instructions

1. Clone this repository.
2. Load the raw Nashville housing dataset into your Oracle database.
3. Execute the provided Oracle SQL cleaning script.
4. Use the cleaned dataset for further analysis or visualizations.
